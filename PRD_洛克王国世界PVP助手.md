## 1. 执行摘要

洛克王国世界PVP助手是一款面向全量玩家的综合型对战辅助Web工具，核心解决玩家在PVP对战中"不知道怎么配队""看不懂当前meta环境""输了不知道输在哪"三大痛点。产品通过公开数据爬取与分析引擎，为玩家提供**阵容推荐、实时环境监控、对战数据分析**三大核心能力，帮助新手快速入门、助力进阶玩家稳步上分。MVP计划在1个月内上线，覆盖宠物图鉴、热门阵容、属性克制等基础模块。

---

## 2. 背景与上下文

### 2.1 问题空间

洛克王国世界作为一款宠物对战类游戏，PVP是其核心玩法之一。然而玩家在进行PVP对战时普遍面临以下问题：

| 痛点 | 影响人群 | 严重程度 |
|------|---------|---------|
| 宠物数量庞大（数百只），新手不知从何练起 | 新手玩家 | ⭐⭐⭐⭐⭐ |
| 版本更新频繁，meta环境变化快，难以跟踪 | 全量玩家 | ⭐⭐⭐⭐ |
| 缺乏可靠的对战数据支持，配队凭感觉 | 进阶玩家 | ⭐⭐⭐⭐ |
| 对局结束缺乏复盘工具，无法针对性提升 | 全量玩家 | ⭐⭐⭐ |
| 各渠道攻略分散、质量参差不齐 | 全量玩家 | ⭐⭐⭐ |

### 2.2 市场机会

洛克王国世界拥有庞大的玩家基础，但市面上缺乏一款**综合型、数据驱动、且保持最新**的PVP辅助工具。现有解决方案多为零散的社区攻略、视频教程或过时的Wiki页面，难以满足玩家实时、一站式的需求。

### 2.3 竞品画布

| 产品类型 | 代表 | 优势 | 劣势 |
|---------|------|------|------|
| 社区攻略帖 | NGA/贴吧 | 玩家真实经验 | 分散、过时、无法搜索 |
| 视频攻略 | B站UP主 | 直观演示 | 实时性差、查找不便 |
| Wiki百科 | 各类Wiki | 资料全面 | 缺乏对战分析维度 |

## 3. 目标与成功指标

### 3.1 产品目标

1. **降低PVP入门门槛** — 让新手玩家在1周内掌握主流阵容和基本对战策略
2. **提供实时Meta环境感知** — 以天为单位更新热门阵容、宠物出场率
3. **建立数据驱动的对战分析能力** — 帮助玩家识别自身弱点，针对性提升

### 3.2 非目标（明确不做）

1. **不提供游戏自动化/脚本功能** — 遵守游戏运营规范，不触犯封禁风险
2. **不做游戏模拟器** — 不开发实际对战模拟功能（受限于数据精度）
3. **不做社交/论坛** — 聚焦工具属性，社区讨论可引导至现有平台
4. **不接入第三方登录** — MVP阶段不做用户系统

### 3.3 成功指标

| 指标 | 当前 | 目标 | 测量方式 |
|------|------|------|---------|
| DAU（日活跃用户） | 0 | MVP上线2周内达500 | 站点统计 |
| 宠物图鉴覆盖率 | 0 | 上线时≥90% | 数据采集覆盖率 |
| 热门阵容日更新率 | 0 | 每天更新 | 定时任务执行率 |
| 用户回访率（7日） | 0 | ≥30% | Cookie/Storage追踪 |
| 页面加载时间 | — | 首屏≤2秒 | Lighthouse监测 |

| 核心诉求 | 实时meta数据、阵容克制分析 |
| 使用场景 | 每次PVP前查环境、输了对局复盘 |
| 技术熟练度 | 中等，能接受数据表格 |

| 维度 | 描述 |
|------|------|
| 年龄 | 16-28岁 |

## 5. 用户故事与需求

### P0 — 必须实现（MVP核心）

| # | 用户故事 | 验收标准 |
|---|---------|---------|
| U001 | 作为玩家，我希望浏览完整的宠物图鉴，查看每只宠物的属性、种族值、技能池，以便了解游戏中的可用宠物 | 1. 按属性、稀有度等维度分类展示 2. 宠物详情页展示基础属性、种族值、可学技能 3. 支持搜索和筛选 |
| U002 | 作为玩家，我希望查看当前版本的**热门阵容推荐**，以便快速了解meta环境 | 1. 展示TOP10热门阵容 2. 每条阵容包含宠物组合、技能搭配 3. 标注胜率/出场率数据 4. 每日自动更新 |
| U003 | 作为玩家，我希望查看**属性克制表**，以便在对战中快速确认克制关系 | 1. 完整的属性克制循环表 2. 支持双向查询（某属性克制谁/被谁克制） |
| U004 | 作为玩家，我希望查看**宠物出场率/胜率排行**，以便了解当前版本强势宠物 | 1. 按出场率、胜率排序 2. 支持按段位筛选 3. 按周/月更新 |
| U005 | 作为玩家，我希望查看每只宠物的**技能详情**，包括威力、PP值、附加效果，以便做技能取舍 | 1. 技能效果完整展示 2. 技能覆盖范围（单/群） 3. 支持按技能类型筛选 |

### 6.2 技术选型建议

| 层面 | 技术方案 | 说明 |
|------|---------|------|
| 前端框架 | React / Vue3 | 单页应用，响应式设计 |
| UI组件库 | Ant Design / Element Plus | 快速搭建，兼顾移动端 |
| 后端框架 | Node.js (Express/Nest) / Python (FastAPI) | 轻量快速开发 |
| 数据库 | PostgreSQL + Redis | 结构化数据+缓存 |
| 数据采集 | Python Scrapy / Puppeteer | 网页爬虫+自动化 |
| 部署 | Vercel / 阿里云ECS | 国内访问优先考虑阿里云 |
| CDN | 阿里云CDN / Cloudflare | 静态资源加速 |

### 6.3 MVP功能范围

**MVP（第1个月）上线内容：**

1. ✅ **宠物图鉴** — 完整的宠物数据库，含属性、种族值、技能池
2. ✅ **热门阵容排行榜** — TOP10阵容，含组合概览
3. ✅ **属性克制表** — 双向查询，可视化展示
4. ✅ **宠物强度排行** — 出场率/胜率排行
5. ✅ **技能数据库** — 完整技能数据查询
6. ✅ **全局搜索** — 宠物、技能搜索

**MVP不做：**
- ❌ 用户系统/登录
- ❌ 阵容构建器（P1）
- ❌ 对战记录上传与分析（P2）
- ❌ 评论/社区功能

### 6.4 数据采集策略

1. **首次采集**（上线前1-2周）：全量爬取宠物数据、技能数据、基础属性，建立完整数据库
2. **日常更新**（上线后）：定时爬取热门阵容、出场率等动态数据，更新频率为每日1次
3. **版本更新响应**：检测到游戏版本更新后，触发增量爬取，标记受影响内容

## 8. 时间线与阶段划分

### 阶段一：MVP (第1-4周)

| 周次 | 任务 | 交付物 |
|------|------|--------|
| 第1周 | 数据采集管道搭建、首批数据入库 | 爬虫模块、初始数据库 |
| 第2周 | 后端API开发、前端框架搭建 | API接口、前端项目骨架 |
| 第3周 | 宠物图鉴页、技能数据页、属性克制表开发 | 三个核心页面完成 |
| 第4周 | 热门阵容排行、搜索功能、整体联调测试上线 | MVP版本上线 |

### 阶段二：增强版 (第5-8周)

- 阵容构建器（拖拽组合+队伍评分）
- 技能配招推荐系统
- 版本更新追踪
- 移动端适配优化

### 阶段三：进阶版 (第9-12周)

- 对战记录上传与分析（P2）
- 用户系统（可选）
- 数据可视化仪表盘改版
- 通知/订阅功能
